Workflow Audit & Batch Operations

555 workflows

Full Workflow Inventory & Naming Audit

Pulled every workflow from the HubSpot instance via the Automation v4 API (555 total, 344 active). Analyzed each workflow's name, trigger type, and function. Identified inconsistent naming patterns across teams and proposed a 3-prefix naming convention: LFC (lifecycle), CMP (campaigns), and SYNC (property sync).
Result

Complete inventory of all 555 workflows with categorization report. Proposed standardized naming convention adopted by the team.

HubSpot Automation v4 API Claude Code Python (pagination + JSON parsing)

Batch Rename 182 Workflows via API

After the audit, batch-renamed 182 workflows to match the new convention. Each rename required reading the full workflow object first (GET), modifying only the name field, then writing it back (PUT) - because HubSpot's v4 PUT API treats omitted fields as deletions. Backed up every workflow's revision history before making changes.
Result

182 workflows renamed in a single session. All workflows backed up with revision snapshots for rollback safety.

HubSpot Automation v4 API Claude Code Python (batch script)

Disaster Recovery from API Incident

During the batch rename, discovered that HubSpot's v4 PUT endpoint destroys workflow actions if you send a partial payload (a behavior not clearly documented). 59 workflows lost their internal actions. Wrote a recovery script that pulled each workflow's previous revision from the revision history API and restored them.
Result

All 59 affected workflows fully restored from revision history. Documented the API behavior to prevent recurrence.

HubSpot v4 Revisions API Claude Code Python (recovery script)

Key learning: HubSpot's v4 PUT endpoint treats omitted fields as "delete." Always GET the full object first, modify only what you need, and PUT the entire object back. This was discovered in production and recovered from programmatically.
