▶ What the skill does under the hood
1. Parses LinkedIn URL or email from your input
2. Calls Amplemarket Reveal API for enrichment data
3. Searches HubSpot contacts by email (dedupe check)
4. If new: POST to HubSpot Contacts API with enriched fields
5. If existing: PATCH to update missing fields only
6. Sets lifecycle stage, lead status, and owner assignment
7. Returns direct link to the HubSpot record
8. Supports batch mode: pass multiple URLs at once

The Time It Went Wrong

Transparency matters. Here's what happened when I wasn't careful enough.

⚠

The HubSpot API Call That Broke 182 Workflows

LESSON LEARNED

HubSpot's v4 PUT API treats omitted fields as "delete." A partial update payload meant to rename workflows wiped all actions from 182 workflows in one call. Recovery took hours using revision history.

What went wrong: Sent a PUT request with only {name, type, isEnabled, revisionId} - the API deleted everything not included in the payload. All workflow actions were gone.
What I built after: Permanent guardrails in Claude Code's memory - it now always GETs the full object before any PUT, modifies only the target field, and sends the complete payload back. The mistake can never happen again because Claude Code remembers it across every future session.
