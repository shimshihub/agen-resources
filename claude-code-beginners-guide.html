What Is Claude Code

The 60-second explanation

Claude Code is Anthropic's AI assistant that runs in your terminal (the command line on your computer). Unlike ChatGPT or the Claude web app, it can:

Read & write files

It sees your project folder. It can create files, edit them, read spreadsheets, generate HTML reports.

Run commands

It executes shell commands - curl API calls, run scripts, install packages, push to GitHub.

Remember context

A CLAUDE.md file in your project gives it permanent memory - your personas, naming conventions, team info.

Build skills

Reusable commands (like /hubspot or /recap) that you invoke anytime. They compound over time.

Think of it this way: ChatGPT is like texting someone smart. Claude Code is like having that smart person sitting at your desk, logged into your tools, with full context on your business.

Install & First Run

5 minutes to get going

Prerequisites

  • A computer - Mac, Linux, or Windows (via WSL)
  • Node.js 18+ - check with node --version in your terminal
  • An Anthropic account - sign up at console.anthropic.com. Claude Code uses the Max plan or API credits.

Install

Terminal
# Install Claude Code globally $ npm install -g @anthropic-ai/claude-code # Navigate to your project folder (or create one) $ mkdir my-gtm-engine && cd my-gtm-engine # Launch Claude Code $ claude Welcome to Claude Code! How can I help you today?
Don't have Node.js? On Mac, install it with brew install node. On Windows, download from nodejs.org. On Linux, use your package manager.

First launch

The first time you run claude, it will ask you to authenticate with your Anthropic account. Follow the prompts - it opens a browser window to log you in.

After that, you're in. Just type naturally and press Enter.

Your First Conversation

Start small, think out loud

Claude Code works through conversation. You describe what you want, it figures out how to do it, and asks for your permission before taking action.

Your first conversation
> Create a folder structure for my marketing project. I need folders for campaigns, content, research, and scripts. I'll create the following structure: campaigns/ content/ emails/ social/ research/ scripts/ Allow? (y/n) y Created 5 directories. > What files do I have in this project? Your project currently has the folder structure we just created. No files yet - just empty directories ready for your content.

Key things to know

  • It asks permission before creating files, running commands, or making changes. You always approve.
  • Talk naturally - you don't need technical language. "Show me my stale HubSpot deals" works.
  • Be specific about what NOT to do - "Read my HubSpot data but don't change anything" is very helpful.
  • Paste anything - URLs, screenshots, error messages, email copy. It handles it all.
Pro tip: When you're exploring something new, start with "just read, don't change anything." Get comfortable with what Claude Code can see before letting it take action.

Connecting to Your CRM

The moment it gets powerful

Claude Code becomes a marketing operations powerhouse once you give it API access to your tools. Here's how to connect HubSpot (same pattern works for any API).

Step A: Get your API key

In HubSpot: Settings > Integrations > Private Apps > Create a private app. Give it read access to contacts, companies, and deals. Copy the access token.

Step B: Store it in a .env file

Setting up your API key
> Create a .env file with my HubSpot API key. The key is: pat-na1-xxxxxxxx Creating .env file... HUBSPOT_API_KEY=pat-na1-xxxxxxxx Created .env file. Added .env to .gitignore so it won't accidentally be committed to git.

Step C: Ask it something

Your first API call
> Using my HubSpot API key from .env, show me my 10 most recent deals. Don't change anything - just read. Loading API key from .env... Calling HubSpot CRM API (GET /crm/v3/objects/deals)... | Deal Name | Stage | Amount | Close Date | |--------------------|-------------|----------|------------| | Acme Corp | Demo | $45,000 | Mar 15 | | Beta Inc | Proposal | $32,000 | Mar 28 | | Gamma Ltd | Negotiation | $78,000 | Apr 10 | | ...
Start read-only. Give your private app only read scopes at first. Once you're comfortable, you can add write access. This prevents accidental changes while you're learning.

Building Your First Skill

This is where the compounding starts

A "skill" is a reusable command you can invoke anytime with a slash. Instead of explaining what you want each time, you teach Claude Code once and it remembers.

How skills work

Creating a skill
> I want to create a reusable skill called "pipeline" that shows me my open HubSpot deals grouped by stage, with total values per stage and deals that haven't been touched in 30+ days flagged as "at risk." Save it as a slash command I can invoke with /pipeline anytime. Creating .claude/commands/pipeline.md... Skill created! You can now run /pipeline anytime.

Using the skill

Running your skill
> /pipeline Querying HubSpot deals API... Pipeline Health Summary: Demo Scheduled | 4 deals | $156,000 Proposal Sent | 3 deals | $89,000 Negotiation | 2 deals | $143,000 At Risk (30+ days stale): - Acme Corp (Demo, $45K) - 38 days - Delta Co (Proposal, $22K) - 45 days

Skills are just markdown files in .claude/commands/. They're instructions that tell Claude Code what to do when you invoke the command. You can edit them anytime.

Skill ideas to start with: /pipeline (deal overview), /stale (needs attention), /weekly-report (activity summary), /campaign-check (campaign status). Start simple, add complexity over time.

The CLAUDE.md File

Your AI's permanent brain

This is the most important file in your project. CLAUDE.md sits in your project root and is loaded into every conversation automatically. It's where you put everything Claude Code should always know.

What to put in it

CLAUDE.md - example structure
# My GTM Engine ## Company Context - Company: Acme Corp - Product: B2B analytics platform - CRM: HubSpot (Portal ID: 12345) - Main competitors: Mixpanel, Amplitude, Heap ## Team - Me (Marketing Ops) - Sarah (Content Lead) - James (AE, owns enterprise deals) - Maria (AE, owns mid-market) ## Campaign Naming Convention Format: [Product]-[Channel]-[Type]-[Date] Example: Analytics-LinkedIn-Demo-2026Q1 ## HubSpot Setup - Pipeline: "Sales Pipeline" (default) - Deal stages: Lead > Demo > Proposal > Closed Won - Custom properties: campaign_source, product_interest ## Personas - Data Engineer (technical buyer) - VP Analytics (business buyer) - CISO (security approver) ## Important Rules - NEVER delete contacts or deals without asking me - Always use the campaign naming convention above - When creating emails, sender should be "Acme" not personal

Every conversation starts with this context loaded. Claude Code knows your team, your naming conventions, your CRM setup, your competitors. You teach it once.

This is the compounding secret. Every time you add context to CLAUDE.md - a new competitor, a learned API quirk, a team change - every future session benefits. After a few weeks, Claude Code knows your business better than most colleagues.

Safety & Best Practices

Learned the hard way so you don't have to

Claude Code is powerful. That means you need guardrails. Here's what I learned from 2 months of daily use - including one painful mistake.

The Golden Rules

  • Start read-only. Give APIs only read access first. Add write access after you're comfortable with how Claude Code behaves.
  • Test on one record first. Before any bulk operation, test on a single contact, deal, or workflow. Confirm it worked. Then go broader.
  • Say what NOT to do. "Analyze these workflows but don't modify anything" is clearer than "look at my workflows."
  • Review before approving. Claude Code asks permission before running commands. Read what it's about to do. Don't auto-approve.
  • Never store API keys in CLAUDE.md. Use .env files and make sure .env is in your .gitignore.
  • Document mistakes in CLAUDE.md. When something goes wrong, add it to your context file so Claude Code never repeats it.
  • Use Plan Mode for big changes. Type /plan to have Claude Code outline its approach before executing. Review the plan, then approve.
Real story: I used a HubSpot PUT API that treats missing fields as "delete." One API call wiped actions from 182 workflows. I recovered using revision history, and now Claude Code has a permanent guardrail in memory: always GET the full object before any PUT. The mistake can never happen again. Lesson: start small, document everything.

Permission levels to consider

Week 1-2: Read Only

Query your CRM, read campaigns, analyze data. No writes. Get comfortable.

Week 3-4: Selective Write

Update properties, create contacts. Always test on one record first.

Month 2+: Full Operations

Create workflows, send emails, manage campaigns. With documented guardrails.

Ongoing: Always Review

No matter how experienced, review bulk operations before approving.

Ideas for Programmatic Marketers

Where to start getting value

Here are concrete things you can do in your first week, roughly in order of complexity.

Day 1 - Quick Wins

  • CRM health check: "Show me all deals with no close date set" or "Which contacts have no owner?"
  • Campaign naming audit: "List all my HubSpot lists and flag any that don't follow our naming convention"
  • Stale deal detection: "Show me deals with no activity in 30+ days, grouped by owner"

Day 2-3 - Content & Reporting

  • Email copy review: Paste your email sequence and ask for issues (broken links, inconsistent branding, compliance gaps)
  • Competitive snapshot: "Research [competitor] and create a one-page comparison for our sales team"
  • Weekly pipeline report: Create a /weekly-report skill that generates a summary HTML

Day 4-5 - Workflow Automation

  • Lead routing analysis: "Read my HubSpot workflows and tell me if there's overlap or gaps"
  • UTM parameter audit: "Check all our landing page URLs for consistent UTM tagging"
  • Contact enrichment: Connect an enrichment API and build a /enrich skill

Week 2+ - Build Your Ops Stack

  • Campaign launch checklist: A skill that validates every campaign before launch (UTMs, lists, attribution, budget)
  • LinkedIn content pipeline: Generate post ideas from your recent work and industry news
  • Multi-channel reporting: Pull data from HubSpot + Google Ads + LinkedIn Ads into one view
  • QBR prep: A skill that generates your quarterly business review deck data
The pattern: Start with a question you ask yourself regularly. Automate it into a skill. Over time, your morning routine becomes: open terminal, run 3-4 slash commands, have full visibility into your business in 2 minutes.

Mindset tips

  • Save everything. Ask Claude Code to save outputs to files. Build a library of reports, analyses, and templates.
  • Invest in CLAUDE.md. 30 minutes building your context file saves hours across every future session.
  • Share what you build. Skills are just markdown files. You can share them with teammates or export them.
The real unlock: Claude Code isn't about doing one thing faster. It's about compounding automation. Every skill you build, every context you add, every guardrail you document - they all stack. After a month, you're operating at a different level.
